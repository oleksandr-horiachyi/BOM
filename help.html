Overview

The problem these tools solve

In a typical engineering team, designers work on individual assemblies — curtain wall panels, structural units, modular components. Each assembly is a collection of parts, and each part has a unique drawing code assigned by the designer.

"What used to take one or two full days now takes 15 minutes."

The designer logs every part code into a LOG file — a structured Excel sheet with all the drawing parameters. When it's time to send work to production, they need to create a Production Schedule: match every part code from every assembly to the LOG, calculate quantities based on how many units of each assembly are needed, and produce a formatted workbook ready for the workshop.

Before this tool existed, that entire process was done manually — copy-pasting in Excel, looking up part codes row by row, recalculating quantities by hand. Every time. For every new Production Schedule.

FabTools automates this completely. Upload your BOM and LOG files, click Run, and download the finished reports in seconds.

💡
Key insight The tools work because everyone on the team uses the same Excel format. Once all designers use the same BOM and LOG structure, the data can be collected, analysed, and processed automatically — across projects, across assemblies, across teams.

⚙️ Tool 1 — BOM Production Automation

This is the core tool. It takes your BOM file (what parts are needed and how many per assembly) and your LOG file (the master list of all drawing codes with their parameters), matches them by part code, and generates two ready-to-use Excel reports.

FileWhat it isFormat
BOM file Bill of Materials — one sheet per assembly. Each sheet lists part codes, descriptions, and quantities. .xlsx / .xlsm
LOG file Production tracking file — one sheet per production unit (e.g. Unit-A, Unit-B or simply 1000, 2000 — the naming convention is up to you). Contains all drawing codes with parameters like length, dimensions, material, and finish. .xlsx / .xlsm
⚠️
Important: the LOG file is not a system log! Despite the name, the LOG file is a regular Excel spreadsheet — your production tracking sheet where each tab represents one production unit. It is not a text file or application log.

Step by step

1
Upload your BOM file

Your multi-sheet Excel BOM. Each sheet = one assembly unit. The tool reads part codes from column A (starting row 3) and quantities from column E.

💡 The number of units per assembly is read from cell E2 of each sheet.
2
Upload your LOG file

Your production tracking Excel file. Each sheet tab is one production unit — you can name them anything (e.g. 1000, 2000, Panel-A, Block-01). Part codes must be in column A starting from row 4.

3
Select LOG sheets

The tool reads all sheet names from your LOG file and shows them as checkboxes. Select which production units you want to include in the Production Schedule.

💡 You can select all or just a subset. For example, if your LOG file has 10 sheets but you only need to schedule 3 units this week, tick only those three.
4
Optionally upload your company logo

If you upload a logo (.png or .jpg), it will appear in the top-left corner of every Production Schedule sheet. This is optional — reports are complete without it.

5
Click Run

The tool matches every part code from your BOM against the selected LOG sheets, calculates total quantities, and builds your reports automatically. Takes a few seconds.

6
Download your reports

Download the Combined BOM, the Production Schedule, or both in a single .zip file. Open directly in Excel — headers styled, columns auto-fitted, filters applied.

🔀 Tool 2 — BOM Merge Tool

Use this tool when you have multiple separate BOM files that need to be combined into one. For example, when different designers have created BOMs for different parts of the same project and you need to consolidate them.

💡
When to use Tool 2 vs Tool 3 Use Tool 2 (Merge) when you want to combine the actual part quantities — matching part codes get their QTY summed together. Use Tool 3 (Combiner) when you want to collect separate assemblies into one BOM file while keeping each assembly as its own sheet.

Color highlights explained

The merged output file uses color coding so you can immediately see what changed and which file it came from:

  • 🟡 Yellow — changes from File #2
  • 🔵 Blue — changes from File #3
  • 🟢 Green — changes from File #4
  • 🩷 Pink — changes from File #5
  • 🟣 Purple — changes from File #6

For updated rows (QTY summed), only the QTY cell is highlighted. For entirely new rows, the full row is highlighted.

Drawing code detection

The Drawing Codes sheet in the Production Schedule groups parts by their base drawing number — stripping the variant suffix (like -1, -2) to find the parent drawing.

Part codeAuto-detected drawing code
10001000 (no suffix)
1000-11000
1000-991000
4187-134187
Panel-A-1Panel-A
Panel-APanel-A (no suffix)
💡
Auto mode rule The tool automatically strips a trailing -N suffix if N is a pure integer with 1–3 digits (i.e. up to 999). This works for most naming conventions without any configuration. If your codes use a different structure, switch to Fixed length mode in the tool settings and specify exactly how many characters make up your base drawing code.
⚠️
Part codes: text vs number Store all part codes as text in Excel, not as numbers. If Excel stores a code like 4003 as a number, it may display as 4003.0 internally, which will prevent matching. To force text format: select the column → Format Cells → Text, then re-enter the values. Or prefix codes with an apostrophe: '4003.

Frequently asked questions

🔒 Are my files safe? What happens to my data?
Your files are processed entirely in a temporary server session and are never stored, logged, or shared. Once you close the browser tab, everything is gone. Anthropic/Streamlit does not have access to your files.
📄 What file formats are supported?
All tools accept .xlsx and .xlsm (macro-enabled) Excel files. The tools do not support .xls (old Excel format), .csv, or .ods files.
⚙️ Do I need to install anything?
Nothing at all. FabTools runs entirely in your web browser. No software installation, no account creation, no license required.
🔢 What if some part codes don't match between BOM and LOG?
The tool automatically generates a Missing parts sheet and a LOG Checks sheet listing every unmatched code. Nothing is silently ignored.
💰 Is it free?
Yes, completely free. No usage limits, no subscription, no hidden costs.
📥 Where can I get sample files to try the tools?
Download sample files from the FabTools homepage:
  • Tool 1: sample-bom.xlsx + sample-log.xlsx
  • Tool 2 & 3: sample-1.xlsx, sample-2.xlsx, sample-3.xlsx, sample-4.xlsx, sample-5.xlsx — upload all five to test Tool 3, or any two/three to test Tool 2
🐛 Something isn't working — what should I do?
First, download the sample files and try running the tool with those. If the sample files work but your own files don't, the most common causes are: wrong column layout, part codes stored as numbers instead of text, or unexpected sheet structure. Check the file format guide above.
